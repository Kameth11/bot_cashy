LO QUE YA FUNCIONABA (MVP original):
------------------------------------
- Bot de Telegram con Telegraf (Node.js)
- Registro de movimientos con regex: "consulta Juan $15000 efectivo"
- CRUD completo: agregar, editar, eliminar, limpiar movimientos
- Reportes: /balance, /hoy, /semana, /mes, /ingresos, /egresos, /pendientes
- Conversion USD/ARS con Bluelytics
- Multi-usuario con Google Sheets separados
- Auth por email + codigo de invitacion
- Confirmaciones (eliminar, editar, cotizacion USD)

LO QUE IMPLEMENTAMOS HOY:
--------------------------
1. INTELLIGENCIA ARTIFICIAL - LENGUAJE NATURAL (NLP)
   - Archivo: index.js (funcion parsearConIA)
   - Modelo: GPT-4o-mini de OpenAI
   - Entiende jerga argentina:
     * "lucas" = 1000 pesos (300 lucas = 300.000)
     * "palos" = 1000 pesos
     * "guita", "mangos" = pesos
     * "le entro a Diego" = ingreso de Diego
     * "credito/debito" = tarjeta
     * "U$", "USD", "verdes" = dolares
   - Flujo: Si el regex no matchea -> intenta con IA -> si no es movimiento, muestra ayuda
   - El regex original sigue andando como fallback

2. LECTURA DE FOTOS DE AGENDA (Vision API)
   - Archivo: index.js (funcion procesarFotoAgenda)
   - Modelo: GPT-4o-mini Vision
   - Flujo: mandas foto -> bot lee los turnos -> confirma -> guarda en tab "Agenda"
   - Crea automaticamente el tab "Agenda" en el Google Sheet
   - Columnas del tab Agenda: Fecha, Hora, Cliente, Servicio, Estado

3. CREACION AUTOMATICA DE TAB AGENDA
   - Funcion: crearTabAgendaSiNoExiste()
   - Si el tab "Agenda" no existe, lo crea con los headers
   - Se ejecuta automaticamente cuando confirmas turnos de una foto

4. OTROS CAMBIOS

================================================================================
                    FLUJOS DE DATOS
================================================================================

FLUJO 1: TEXTO (Lenguaje natural)
  Usuario escribe texto
    -> No es comando "/" ?
      -> Hay estado pendiente? (editar, eliminar, etc.) -> manejar estado
      -> No hay estado pendiente:
        -> Intentar regex (formato: "consulta Juan $15000 efectivo")
          -> Match? -> Registrar movimiento
          -> No match?
            -> Intentar IA (parsearConIA)
              -> Es movimiento? -> Registrar con datos de la IA
              -> No es movimiento? -> Mostrar ayuda

FLUJO 2: FOTO (Agenda)
  Usuario manda foto
    -> Bot descarga la imagen de Telegram
    -> Manda a OpenAI Vision API
    -> Extrae turnos (hora, cliente, servicio)
    -> Muestra resumen al usuario
    -> Usuario confirma ("si")
      -> Crea tab "Agenda" si no existe
      -> Guarda turnos en Google Sheets

FLUJO 3: REGEX (Original - sigue andando)
  "consulta Juan Perez $15000 efectivo"
    -> Regex match -> Registrar directamente (sin IA)

COSTO ESTIMADO DE OPENAI:
  - GPT-4o-mini texto: ~$0.15 / 1M input tokens, ~$0.60 / 1M output
  - GPT-4o-mini vision: ~$0.15 / 1M input tokens
  - Cada mensaje de texto: ~$0.0001 (0.01 centavos)
  - Cada foto: ~$0.001 (0.1 centavos)
  - Mensualmente con 500 msgs + 50 fotos: menos de $1 USD/mes
